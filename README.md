# SDN Controller over AmneziaWG Tunnels

A Software-Defined Networking prototype that creates and manages overlay networks using **WireGuard** and **AmneziaWG** tunnels with DPI-resistant obfuscation.

## Features

- **Topology Management** — Define networks as YAML: nodes, tunnels, policies, obfuscation settings
- **Key Management** — Curve25519 key generation, distribution, and rotation
- **Config Generation** — Outputs WireGuard (wg-quick), AmneziaWG, and JSON formats
- **Policy Engine** — Zero-trust access control with priority-based allow/deny rules
- **REST API** — 15+ endpoints for full programmatic control
- **CLI Tool** — `sdnctl` for one-shot management operations
- **Node Agent** — Per-node daemon for config application and health monitoring
- **AmneziaWG Support** — Junk packet injection, handshake obfuscation, header masking

## Supported Topologies

| Topology | Use Case | Nodes | Tunnels |
|----------|----------|-------|---------|
| **Mesh** | Distributed services, multi-region replication | 4 | 6 |

| **Simple Pair** | Development, testing, P2P links | 2 | 1 |

## Quick Start

```bash
# Install dependencies
pip install -r requirements.txt

# Run the comprehensive demo (no root needed)
python3 demo.py

# Generate deployable configs from a topology
python3 cli/sdnctl.py -t examples/mesh.yaml config generate --output-dir ./configs

# Generate AmneziaWG configs with obfuscation
python3 cli/sdnctl.py -t examples/mesh.yaml config generate --format amneziawg

# Start the REST API
python3 cli/sdnctl.py -t examples/hub_spoke.yaml server --port 8080
```

## Example: Mesh Topology

```yaml
# examples/mesh.yaml
name: mesh-network
network_cidr: 10.0.0.0/8
obfuscation_enabled: true

nodes:
  - name: node-nyc
    endpoint: 198.51.100.10:51820
    address: 10.10.1.1/32
    allowed_ips: [10.10.1.0/24]

  - name: node-lon
    endpoint: 203.0.113.20:51820
    address: 10.10.2.1/32
    allowed_ips: [10.10.2.0/24]

tunnels:
  - node-nyc:node-lon   # shorthand syntax

policies:
  - name: allow-all
    source: 10.0.0.0/8
    destination: 10.0.0.0/8
    action: allow
```

Generated WireGuard config for `node-nyc`:

```ini
[Interface]
PrivateKey = <auto-generated>
Address = 10.10.1.1/32
ListenPort = 51820
MTU = 1420

JunkPacketCount = 5       # AmneziaWG obfuscation
JunkPacketMinSize = 40
JunkPacketMaxSize = 80

[Peer]
PublicKey = <peer-public-key>
Endpoint = 203.0.113.20:51820
AllowedIPs = 10.10.2.1/32, 10.10.2.0/24
PersistentKeepalive = 25
```

## REST API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/health` | Health check |
| GET | `/api/v1/status` | Topology summary |
| GET | `/api/v1/topology` | Full topology export |
| GET/POST | `/api/v1/nodes` | List/add nodes |
| GET/DELETE | `/api/v1/nodes/<name>` | Get/remove node |
| GET/POST | `/api/v1/tunnels` | List/create tunnels |
| GET/POST | `/api/v1/policies` | List/add policies |
| GET | `/api/v1/configs/generate` | Generate all configs |
| GET | `/api/v1/configs/<node>` | Get node config (wg-quick/amneziawg/json) |
| POST | `/api/v1/evaluate` | Evaluate flow against policies |

## AmneziaWG vs WireGuard

AmneziaWG is a WireGuard fork that adds protocol obfuscation to bypass Deep Packet Inspection. The controller supports both:

- **Standard WG**: Clean protocol, no obfuscation — best for trusted networks
- **AmneziaWG**: Junk packets, header masking, handshake obfuscation — for DPI-heavy environments

Set `obfuscation_enabled: true` in the topology YAML and configure per-node parameters.
